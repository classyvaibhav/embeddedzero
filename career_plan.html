0310-Week Learning Plan

Four phases balancing basic electronics · embedded hardware · embedded software. Study 4–5 hrs/day weekdays + 6 hrs Saturday. Sunday off. No bare-metal firmware rabbit holes — everything stays at the application level for an entry-level embedded role.

Phase 1 — Basic Electronics Fundamentals
Weeks 1–2

Core Concepts (rebuild intuition)

  • Ohm's law + Kirchhoff's laws — solve 20 numerical problems
  • Voltage & current dividers — when to use them (sensor biasing, level shifting)
  • RC & RL circuits — charging / discharging, time constant intuition
  • Diodes & LEDs — forward drop, current-limit resistor calculation
  • Transistor as a switch (BJT & MOSFET) — driving a relay / motor
  • Resources: ALL ABOUT CIRCUITS free textbook + Ben Eater YouTube

Op-Amps & Digital Logic

  • Op-amp basics: inverting, non-inverting, buffer, comparator
  • Why op-amps matter: sensor signal conditioning
  • Digital logic: AND / OR / NOT / NAND / XOR, flip-flops, counters
  • Simulate in Tinkercad Circuits (free, browser) or Falstad

Profile Setup + Apply-Back

  • Customise LinkedIn URL to bahar-jindani
  • Create GitHub with profile README
  • LinkedIn headline: "MSc Applied Physics | Embedded Systems (HW + SW) & Electronics Testing"
  • Apply to Aimtron full-time — this week, while the cert ink is fresh
Phase 2 — Schematic, Datasheets, Components & BOM
Weeks 3–4

Schematic Reading

  • Read 15–20 open-source schematics (Arduino Uno, ESP32 DevKit, SparkFun/Adafruit boards)
  • Identify blocks: power, MCU, reset, crystal, sensors, I/O, connectors
  • Symbol library: R, C, L, diode, MOSFET, BJT, op-amp, regulator
  • Resource: SparkFun "How to Read a Schematic" + EEVblog #170

Datasheet Reading & Component Selection

  • Navigate a datasheet: pinout, absolute max ratings, electrical characteristics
  • Read 10 datasheets across: MCU, sensor, regulator, op-amp, MOSFET
  • Given a requirement (e.g., "LDO 5V→3.3V, 500mA"), pick a real part
  • Sources: DigiKey, Mouser, Robu.in, Octopart for cross-referencing
  • Resource: Phil's Lab YouTube, DigiKey Learning Center

BOM Creation

  • Standard BOM columns: Designator, Part, Value, Footprint, Manufacturer, MPN, Alt. MPN, Qty, Price
  • Create BOM for a real design (Arduino clone, ESP32 sensor node)
  • Use Excel / Google Sheets, export as CSV
  • This becomes Project 1 (see Projects section)
Phase 3 — Embedded Software + Hands-On Build
Weeks 5–7

Arduino / ESP32 Coding

  • Buy ESP32 DevKit (₹400–600) + Arduino Uno (₹500) + breadboard kit
  • Arduino IDE + C/C++ basics (variables, arrays, functions, structs — enough to read & modify code)
  • GPIO, ADC, PWM, timers, interrupts via standard libraries
  • Writing code: read an example → modify → combine features. That's 90% of entry-level embedded work.

Protocols via Libraries

  • UART with Serial; I2C with Wire.h; SPI with SPI.h
  • Interface real sensors: DHT11 (temp/humidity), BMP280 (pressure), 0.96" OLED (I2C)
  • Goal: talk to a new sensor by reading its datasheet + library docs
  • Simulator fallback: Wokwi (free, browser-based)

Circuit Build + Debug

  • Build 2 real breadboard circuits (Projects 2 + 3)
  • Debug with multimeter + oscilloscope (she has lab access)
  • Common bugs: wrong pin, floating inputs, missing pull-ups, wrong logic levels
  • Document debugging process in GitHub README — recruiters love this
Phase 4 — Light PCB + Apply & Interview
Weeks 8–10+

PCB Basics (read & modify, not design from scratch)

  • KiCad free — open any ESP32 reference design, understand schematic + layout
  • Do a small modification (swap a component, add a header) — that's enough for entry-level
  • Export Gerber files once as an exercise
  • Optional: Altium Viewer (free) — most Indian EMS use Altium; familiarity helps

Application Blitz

  • 15 applications/day on Naukri, LinkedIn, foundit
  • Company-direct portals (see Target Companies)
  • 10 cold LinkedIn DMs/day to hiring managers
  • MSU Baroda alumni referral outreach — highest conversion channel

Interview + Backup

  • 2 technical + 1 HR mock per week; record on phone
  • Practise: "walk me through this schematic", "pick a resistor for this LED", "what's in your BOM?"
  • Backup channels: TCS NQT, Infosys, Wipro fresher drives
  • Eduskills-AICTE virtual internship — certified + placement assistance

07Interview Preparation

Hardware Engineer Trainee interviews stay within a predictable scope: circuit reading, components, datasheets, BOM, Arduino-level embedded. Master these and she clears 80% of entry-level screens — no deep-firmware rabbit holes needed.

Technical — Must-Know (HW + SW + Basic Electronics)

  • Basic electronics: Ohm's law, voltage divider formula, RC time constant — derive on paper
  • Read a resistor colour code on the spot
  • Pick current-limit resistor for an LED given Vf and If
  • Op-amp: explain inverting, non-inverting, buffer, comparator
  • Difference between BJT and MOSFET; when to use which as a switch
  • Explain a PN junction — forward & reverse bias (physics MSc wins here)
  • How do you pick a pull-up / pull-down resistor value? Why 10k?
  • How do you pick a decoupling capacitor? Why 100nF ceramic near every IC power pin?
  • LDO vs switching regulator — when to pick each
  • Walk through this schematic — name each block (interviewer will show one)
  • How do you approach a datasheet? Which sections matter most?
  • Walk me through a BOM you created — why these parts, why these alternates?
  • How do you source components via DigiKey / Mouser / Octopart?
  • UART vs I2C vs SPI — wiring, speed, addressing, when to pick each
  • How does ADC work? Resolution vs sampling rate
  • What is PWM? How is duty cycle controlled?
  • Read this Arduino code snippet and explain what it does (interviewer will show)
  • How would you interface a new sensor you've never used? Walk through your process
  • Explain your Aimtron work — 90-second walkthrough
  • Explain Laplacian and Sobel image-sharpness detection (Aimtron calibration)
  • Walk through one GitHub project — architecture, components, code, decisions

HR — Must-Answer Smoothly

  • Walk me through your resume (90 sec, rehearsed)
  • Why Electronics/Embedded for a Physics MSc?
  • Why should we hire you over a BTech ECE candidate?
    → Answer: physics depth for root-cause analysis + Aimtron industry exposure
  • What did you learn at Aimtron? (Have 3 specific stories)
  • Are you open to relocation? (Bangalore/Pune/Chennai)
  • Are you open to shift work? (EMS production lines run 24x7)
  • Salary expectation: "Based on entry-level market rates ₹25–32k aligns; open to full-package discussion"
  • 5-year plan: "Grow into a senior test engineer / embedded architect role" (avoid "PhD")
  • Weakness: pick a real one with a fix (e.g., "first-time job interviews — practising mocks")
  • Why this company specifically? (Research 3 facts before each interview)
  • Do you have any questions? (Always have 2 ready)
  • Conflict / teamwork story — prepare a STAR-format answer
  • When can you join? (Know the answer precisely)
  • Any other offers in hand? (Be honest, create healthy tension)
  • Ask about: mentorship, training program structure, team size
