Your First Arduino: Blink!

The "Hello World" of embedded systems. Install the tools. Upload a sketch. Make an LED blink. You're officially an embedded developer after this lesson.

By the end of this lesson, you will:
  • Have Arduino IDE installed with ESP32 board support
  • Upload the Blink sketch and watch the LED blink
  • Understand setup() and loop() — the two functions in every sketch

1Install the Tools

Step 1: Install Arduino IDE

  1. Go to arduino.cc/en/software
  2. Download Arduino IDE 2.x for your OS (Windows / Mac / Linux)
  3. Install it. Open the IDE. You should see an empty sketch window.

Step 2: Add ESP32 Board Support (if you have an ESP32)

  1. Open Arduino IDE → File → Preferences
  2. Find "Additional Boards Manager URLs" → paste:
    https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
  3. Click OK.
  4. Open Tools → Board → Boards Manager. Search "esp32". Install "esp32 by Espressif Systems".
  5. After install, go to Tools → Board → ESP32 Arduino and pick "ESP32 Dev Module".
💡 Skip This If… If you don't have an ESP32 yet, you can still follow along using the Arduino Uno simulation in Wokwi or Tinkercad. Every lesson will show both paths.

2The Classic Blink Sketch

Arduino IDE → File → Examples → 01.Basics → Blink. You'll see this code:

void setup() {
  // runs once when the board is reset / powered on
  pinMode(LED_BUILTIN, OUTPUT);
}

void loop() {
  // runs forever after setup()
  digitalWrite(LED_BUILTIN, HIGH);    // LED on
  delay(1000);                        // wait 1 second
  digitalWrite(LED_BUILTIN, LOW);     // LED off
  delay(1000);                        // wait 1 second
}

Understanding Every Line

  • void setup() { ... } — runs ONCE when power is applied or the board resets. Use for one-time initialisation.
  • pinMode(LED_BUILTIN, OUTPUT); — tells the MCU that this pin should send signals out (instead of reading them in). LED_BUILTIN is a special constant (pin 13 on Uno, pin 2 on ESP32).
  • void loop() { ... } — runs FOREVER after setup. This is your main program. Everything the MCU does happens here.
  • digitalWrite(pin, HIGH); — makes the pin output 3.3V (or 5V on Uno). LOW makes it 0V.
  • delay(1000); — pause for 1000 milliseconds (1 second). The CPU does nothing else during this time.
💡 The Mental Model When you click "Upload", your PC compiles this code to machine instructions for the MCU, sends them over USB to the MCU's Flash memory, then resets the MCU. The MCU runs setup() once, then loop() over and over forever. That's it. Every embedded program in the world follows this pattern.

3Upload to a Real Board

If you have the physical hardware (Arduino Uno or ESP32 DevKit), let's put the code on real silicon.

  1. Connect the board via USB to your laptop
  2. In Arduino IDE: Tools → Board → (Arduino Uno or ESP32 Dev Module)
  3. In Arduino IDE: Tools → Port → select the COM port (usually COM3/COM4 on Windows, /dev/ttyUSB0 on Linux)
  4. Paste the Blink code (or open Examples → 01.Basics → Blink)
  5. Click the → Upload button (right arrow icon, top-left)
  6. Wait ~15 seconds. You'll see "Done uploading" at the bottom
  7. 🎉 The onboard LED (usually labelled "L" or "D13") starts blinking every second!
⚠ Common Issues
  • Wrong COM port: unplug the board, check which ports appear; plug in, see which one shows up — that's yours.
  • Driver missing (Windows): ESP32 boards need the CP210x driver. Download from Silicon Labs.
  • "Failed to connect to ESP32": some ESP32 boards need you to hold BOOT button while the upload starts. Release after "Connecting..." appears.
  • Upload succeeds but nothing happens: wrong board type selected. Double-check Tools → Board.

4Modify & Experiment

Now that Blink works, try these variations. Each is a tiny change to the code. The muscle of "change code → upload → observe" is what makes you an embedded developer:

Experiment 1: Faster Blink

delay(100);   // change 1000 to 100 — 10 Hz blinking

Experiment 2: Morse Code SOS

void loop() {
  // S (three short)
  for (int i = 0; i < 3; i++) {
    digitalWrite(LED_BUILTIN, HIGH); delay(200);
    digitalWrite(LED_BUILTIN, LOW);  delay(200);
  }
  delay(300);
  // O (three long)
  for (int i = 0; i < 3; i++) {
    digitalWrite(LED_BUILTIN, HIGH); delay(600);
    digitalWrite(LED_BUILTIN, LOW);  delay(200);
  }
  delay(300);
  // S again
  for (int i = 0; i < 3; i++) {
    digitalWrite(LED_BUILTIN, HIGH); delay(200);
    digitalWrite(LED_BUILTIN, LOW);  delay(200);
  }
  delay(1500);
}

Experiment 3: Serial Monitor

The Serial Monitor lets your MCU print messages to your PC. This is the #1 debugging tool.

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
  Serial.begin(115200);             // start serial at 115200 baud
  Serial.println("Bahar's first Arduino sketch!");
}

int count = 0;

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  Serial.print("Blink #"); Serial.println(++count);
  delay(500);
  digitalWrite(LED_BUILTIN, LOW);
  delay(500);
}

After uploading: click the Serial Monitor icon (top-right). Set baud rate to 115200. Watch your messages appear!

🔍 Check Yourself

What runs first — setup() or loop()?
setup() — it runs ONCE at boot. Then loop() runs forever.
How many times does loop() execute?
Forever — until you power off or press reset. Every time it returns, it gets called again immediately.
If I want the LED to blink every 250 ms, what do I change?
Change both delay(1000) calls to delay(250).
What does pinMode(5, OUTPUT) do?
Configures GPIO pin 5 as an output so it can drive voltage to external devices (LEDs, etc.). Without this, the pin defaults to high-impedance input.
My Serial Monitor shows gibberish. What's wrong?
Baud rate mismatch. Serial.begin(115200) in code must match the Serial Monitor dropdown (bottom-right). Set both to 115200.

Key Takeaways

  • Every Arduino sketch has two functions: setup() (runs once) and loop() (runs forever).
  • pinMode() configures whether a pin is input or output.
  • digitalWrite(pin, HIGH/LOW) sets the output voltage.
  • delay(ms) pauses for milliseconds.
  • Serial.begin() + Serial.println() — your #1 debugging tool.
  • Always push code to GitHub. Even tiny sketches. Build your portfolio from day one.
