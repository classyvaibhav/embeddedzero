The Analog World — ADC & PWM

Digital pins are just HIGH or LOW. But sensors give smooth voltages, and LEDs can fade at any brightness. ADC and PWM bridge the digital and analog worlds.

By the end of this lesson, you will:
  • Read a potentiometer with analogRead() and convert to volts
  • Fade an LED smoothly with PWM
  • Understand why PWM doesn't "really" output analog voltage

1ADC — Reading the Analog World

Most sensors (temperature, pressure, light, sound) output a continuously-varying voltage. Your MCU's CPU only understands numbers. The Analog-to-Digital Converter (ADC) bridges them: input voltage in, integer number out.

BoardResolutionRangeMapping
Arduino Uno10-bit0 – 10230V → 0; 5V → 1023
ESP3212-bit (configurable)0 – 40950V → 0; 3.3V → 4095
Volts = raw × Vref / (2N − 1)

Example: ESP32 reads raw = 2048. Volts = 2048 × 3.3 / 4095 = 1.65 V (halfway).

💡 The Classic First Experiment Wire a 10k potentiometer: outer legs to Vcc and GND, wiper (middle leg) to an ADC pin. As you turn the knob, you'll see the ADC value sweep from 0 to max. This is the fundamental input pattern for every sensor you'll ever read.

⚠ ESP32 ADC Gotchas
  • ESP32 has ADC1 (GPIO 32–39) and ADC2 (GPIO 0, 2, 4, 12-15, 25-27). ADC2 shuts down when Wi-Fi is active. Use ADC1 pins.
  • Readings are noisy. Average 10 samples for clean results.
  • Nonlinear near 0V and near Vref. Avoid the extreme ends.

2PWM — Fake Analog Output

GPIO pins only output HIGH or LOW. But what if we switch them very fast, spending 30% of the time HIGH and 70% LOW? The average voltage is 30% of Vcc. If a load has any inertia (an LED filament, a motor, a speaker), it responds to the average — so the load "sees" an analog voltage.

This is Pulse Width Modulation (PWM). The fraction of time spent HIGH is the duty cycle:

Vavg = Vcc × (duty / 255)
Duty cycle controls average voltage. 0% = 0V; 100% = full Vcc.

3Fading an LED with PWM

Arduino Uno: analogWrite(pin, 0..255). Works on pins 3, 5, 6, 9, 10, 11. On ESP32, same function works, or use ledcWrite() for more control.

const int LED = 9;

void setup() { pinMode(LED, OUTPUT); }

void loop() {
  // Fade up
  for (int d = 0; d <= 255; d++) {
    analogWrite(LED, d);
    delay(10);
  }
  // Fade down
  for (int d = 255; d >= 0; d--) {
    analogWrite(LED, d);
    delay(10);
  }
}

💡 Why PWM for Motors? PWM is how you vary motor speed without wasting energy as heat. An analog voltage controller (like a potentiometer) would dissipate power. PWM just switches the MOSFET on/off rapidly — the motor averages it, and efficiency stays ~95%.

🔍 Check Yourself

ESP32 ADC with Vref=3.3V, 12-bit. Raw=3000. Voltage?
V = 3000 × 3.3 / 4095 = 2.42V.
Can ESP32 read 5V directly?
No. ADC max is 3.3V. Use a voltage divider (from Lesson 2!) to scale 5V down to 3.3V.
PWM duty=64 on 8-bit, 3.3V supply. Average voltage?
64/255 × 3.3 = 0.83V.
Why use PWM to dim an LED instead of a variable resistor?
PWM is more efficient (no wasted heat), gives wider dynamic range, and doesn't shift the LED's colour (which a variable resistor slightly does at low currents).

Key Takeaways

  • ADC converts analog voltage → integer. Arduino 10-bit (0–1023); ESP32 12-bit (0–4095).
  • ESP32: use ADC1 pins (32–39). ADC2 fights with Wi-Fi.
  • PWM fakes analog output by switching HIGH/LOW fast. Duty cycle = fraction of time HIGH.
  • Arduino: analogWrite(pin, 0..255) on PWM-capable pins (3, 5, 6, 9, 10, 11).
  • Used for: LED brightness, motor speed, audio tones, servo control.
