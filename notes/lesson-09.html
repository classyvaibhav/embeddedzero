Talking to Things — Serial, I2C, SPI

The three communication protocols that power 99% of embedded systems. Send messages to your PC. Read a pressure sensor. Drive an OLED display.

By the end of this lesson, you will:
  • Use Serial to debug any sketch with printed messages
  • Connect an I2C sensor and read data from it
  • Know when to pick SPI over I2C (faster, more pins)

1UART / Serial — Your Best Friend

Serial (UART) is the simplest protocol. Two wires: TX (transmit) and RX (receive), plus a common GND. Both sides agree on a baud rate (9600, 115200 bps typical). That's it.

You already used Serial in Lesson 6 (Serial.println()). Let's go deeper.

void setup() {
  Serial.begin(115200);
  Serial.println("Ready!");
}

void loop() {
  if (Serial.available()) {                  // PC sent a message?
    String cmd = Serial.readStringUntil('\n');
    Serial.print("You sent: ");
    Serial.println(cmd);
  }
  delay(10);
}

Upload, open Serial Monitor, type a message, press Enter. The MCU echoes it back. You've just had a two-way conversation with your microcontroller.

💩 Debug Like a Pro Whenever something's wrong with your code, sprinkle Serial.println() statements everywhere. Print values, flags, "reached here" markers. This is how every embedded engineer debugs — even at companies like Omnidya and Tesla.

2I2C — Many Devices on Two Wires

I2C (pronounced "I-squared-C") lets you connect multiple sensors and chips to just two shared wires: SDA (data) and SCL (clock). Each device has a unique 7-bit address so the master (your MCU) knows who to talk to.

Multiple devices share the same two wires. Each has a unique address on the bus.

I2C Scanner — Find Connected Devices

#include <Wire.h>

void setup() {
  Serial.begin(115200);
  Wire.begin();                  // default SDA=21, SCL=22 on ESP32
  Serial.println("Scanning I2C bus...");

  for (byte addr = 1; addr < 127; addr++) {
    Wire.beginTransmission(addr);
    if (Wire.endTransmission() == 0) {
      Serial.printf("Found device at 0x%02X\n", addr);
    }
  }
}

void loop() {}

3SPI — Faster, More Wires

SPI uses 4 wires per slave: MOSI, MISO, SCK, CS. No addresses — each slave has its own Chip Select line. Much faster than I2C (tens of MHz vs 1 MHz max).

💡 When to Pick Which
  • UART: talking to PC, GPS modules, radio modules.
  • I2C: sensors (DHT22, BMP280, MPU6050), small displays, EEPROMs. Pick when you want many devices on few wires.
  • SPI: SD cards, fast sensors, big displays. Pick when you need speed.

🔍 Check Yourself

What's the minimum number of wires for an I2C connection?
3: SDA, SCL, and GND. (Plus Vcc to power the slave, so 4 total typically.)
Why does I2C need pull-up resistors?
I2C lines are "open-drain" — devices can only pull them low. The pull-ups (typically 4.7k to Vcc) return lines to HIGH when no one is pulling them down.
You need to connect 3 SPI devices. How many CS lines?
3 — one per device. They can share MOSI, MISO, SCK but need separate CS lines.
Serial Monitor shows garbage characters. What's wrong?
Baud rate mismatch between Serial.begin(X) in code and the Serial Monitor's baud dropdown. Set both to same value (usually 115200).

Key Takeaways

  • UART / Serial: 2 wires, simple, slow. Use for PC debug.
  • I2C: 2 shared wires, up to ~127 devices. Pull-ups required. Slower.
  • SPI: 4 wires per slave, fast. One CS line per device.
  • Arduino libraries: Serial, Wire, SPI.
  • Sensor libraries wrap all this complexity — you just call sensor.readTemperature().
