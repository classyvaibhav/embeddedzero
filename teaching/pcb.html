T4.1

PCB Basics — Layers, Traces, Vias, Copper Pour

Session 37 · 40 min
Goal: Look at any 2-layer PCB and identify traces, vias, ground pour, silkscreen, footprints.

A PCB is a sandwich: FR4 substrate + copper layers + solder mask + silkscreen. Most entry-level designs are 2-layer.

  • Traces: copper paths carrying signals/power.
  • Vias: plated holes connecting layers.
  • Pads: where components solder on.
  • Silkscreen: white text labels on top/bottom.
  • Solder mask: green/black coating preventing solder bridges.
  • Ground plane: large copper area tied to GND. Noise + return paths.
Top-to-bottom: silkscreen → soldermask → copper → FR4 substrate → copper → soldermask.
Teach this way
  1. Give her an ESP32 DevKit + magnifier. Point to pads, vias, silkscreen, back-side copper pour.
  2. Open KiCad. Download an ESP32 breakout project from GitHub.
  3. Toggle layers: hide silkscreen, hide mask, see bare copper. Makes it tangible.
  4. Goal is reading, not designing from scratch.
Verify understanding
  1. What's a via? Plated hole connecting layers
  2. Why ground pour? Noise + return paths
  3. Why silkscreen white? Contrast on green mask
T4.2

KiCad Walkthrough — Schematic → Layout → Gerber

Session 38 · 90 min
Goal: Open an existing KiCad project, understand workflow, export Gerbers.

Schematic editor (draw) → Symbol-to-footprint mapping → PCB editor (place + route) → Design Rule Check → Gerber export.

Don't teach from-scratch design. Just familiarity to read PCB projects and make small changes.

Teach this way
  1. Install KiCad 8.x (free).
  2. Clone a simple ESP32 breakout from GitHub.
  3. Schematic editor first — she already knows schematics from Part 2.
  4. PCB editor: layer panel, footprints, traces, ground fill.
  5. Run DRC — shows trace/spacing violations.
  6. File → Plot → Gerber. Show the .gbr files — what a PCB fab gets.
  7. Homework: modify a silkscreen label, re-export. Done — enough for entry-level.
Reference: Shawn Hymel — KiCad Tutorials · KiCad Docs

T4.4

Mock Interview — Run Her Through a Real One

Weekly Session 40+ · 45 min each
Goal: She handles a 30-min technical + 15-min HR without freezing.
Structure each mock
  1. 5 min HR warm-up: Tell me about yourself. Why this role? Any questions?
  2. 15 min schematic / circuit: Show open-source schematic. Pick a block. Pick R for LED. Read datasheet snippet.
  3. 10 min protocol / code: I2C vs SPI? Walk a sketch. What's volatile?
  4. 5 min project walkthrough: Aimtron story + 1 GitHub project. 2-min STAR-format pitch.
  5. 5 min feedback: What went well, what to work on. Record her phone for replay.

Do this weekly from Week 7. Compounds quickly.
